# Google Tasks Telegram Monitor Bot 🤖📋

Асинхронный Telegram-бот на базе **Aiogram 3** и **Google Tasks API**, предназначенный для мониторинга, синхронизации и отображения задач пользователей Person1 и Person2 в реальном времени с выводом уведомлений в групповой чат Telegram.

---

## ✨ Основные возможности

*   🔄 **Двухпользовательский мониторинг:** Одновременное отслеживание списков задач Google Tasks для двух пользователей ("Person1" и "Person2").
*   🔔 **Уведомления в реальном времени:** Оповещения в Telegram-группу (с поддержкой конкретных топиков/тредов) при создании новых задач, изменении существующих или их завершении.
*   🤫 **«Тихий старт» (Quiet Sync):** При первом запуске бот считывает текущее состояние задач и сохраняет его в базу данных без спама и массовой отправки старых уведомлений.
*   💾 **Локальная БД SQLite:** Использует асинхронную библиотеку `aiosqlite` для хранения хешей и времени последнего обновления задач (`tasks_state.db`), чтобы избегать дублирования сообщений.
*   🗓 **Командный интерфейс:**
    *   `/p1_week` — задачи Person1 на текущую неделю.
    *   `/p2_week` — задачи Person2 на текущую неделю.
    *   `/p1_nextweek` — задачи Person1 на следующую неделю.
    *   `/p2_nextweek` — задачи Person2 на следующую неделю.
*   🎨 **Красивое форматирование:** Задачи группируются по дням недели (с переводом на русский язык и учетом часового пояса UTC+9 (корея)). Выполненные задачи отображаются зачеркнутыми (~~задача~~).

## 📂 Структура проекта

```text
TgBot_GoogleTasks/
├── bot/
│   ├── __init__.py

├── database/

│   └── db_manager.py      # Асинхронное взаимодействие с SQLite (aiosqlite)

│   └── google_tasks.py    # Подключение к Google Tasks API
├── auth.py                # Скрипт первоначальной OAuth авторизации в Google
```

## 🚀 Инструкция по настройке и запуску

### 1. Подготовка Google Cloud Console

1.  Перейдите в [Google Cloud Console](https://console.cloud.google.com/).
2.  Создайте новый проект (или выберите существующий).
3.  В разделе **API & Services** включите **Google Tasks API**.
4.  Перейдите в раздел **OAuth consent screen** (Экран согласия OAuth):
    *   Выберите тип **External**.
    *   Укажите название приложения и контактный email.
    *   В разделе **Test users** обязательно добавьте Google-аккаунты пользователей (Person1 и Person2), чьи задачи будут отслеживаться.
5.  Перейдите во вкладку **Credentials** (Учетные данные):
    *   Нажмите **Create Credentials** -> **OAuth client ID**.
    *   Выберите тип приложения **Desktop app** (Десктопное приложение).
    *   Скачайте полученный файл авторизации JSON, переименуйте его в `credentials.json` и положите в корень проекта.
6.  Переведите проект в режим **Production** (Производственный) иначе токены будут сгорать через неделю.


---

### 2. Получение токенов пользователей

Для работы бота в фоновом режиме требуются сохраненные авторизационные токены пользователей. 
Можно воспользоваться другим моим репозиторием специально созданный для получения токена с красивым веб-интерфейсом: https://github.com/KakabegovAtash/TgBot_GoogleTasks_AuthForToken

Если решите вручную получать, без репозитория описанного выше, то нужно получить по очереди:

1.  Убедитесь, что в корне лежит актуальный `credentials.json`.
2.  Запустите скрипт `auth.py`:
    ```bash
    python auth.py
    ```
3.  В браузере откроется окно авторизации Google. Войдите под аккаунтом **первого** пользователя (например, Person1), предоставьте необходимые разрешения.
4.  После успешного завершения в корне создастся файл `token.json`. Переименуйте его в `Person1_token.json`.
5.  Повторите процедуру для **второго** пользователя (Person2):
    *   Снова запустите `python auth.py`.
    *   Авторизуйтесь под аккаунтом Person2.
    *   Полученный `token.json` переименуйте в `Person2_token.json`.

---

### 3. Установка зависимостей и настройка окружения

1.  Создайте виртуальное окружение и активируйте его:
    ```bash
    python -m venv venv
    source venv/bin/activate  # Для Linux/macOS
    # или
    venv\Scripts\activate     # Для Windows
    ```
2.  Установите необходимые пакеты:
    ```bash
    pip install -r requirements.txt
    ```
3.  Создайте файл `.env` на основе шаблона `.env.example`:
    ```bash
    cp .env.example .env
    ```
4.  Заполните переменные в `.env`:
    ```ini
    TELEGRAM_BOT_TOKEN=your_telegram_bot_token
    GROUP_CHAT_ID=-1001234512340         # ID группы Telegram, куда бот отправляет уведомления
    MESSAGE_THREAD_ID=123                # ID топика/треда в группе (если включены темы)
    POLLING_INTERVAL=60                  # Интервал опроса Google Tasks в секундах
    ```

---

### 4. Запуск бота

Для запуска основного цикла мониторинга и Telegram-бота выполните команду в корне проекта:
```bash
python main.py
```

При первом запуске бот:
1.  Инициализирует локальную базу данных `tasks_state.db`.
2.  Выполнит начальную (тихую) синхронизацию всех задач обоих пользователей, чтобы не дублировать старые записи в чат.
3.  Отправит сообщение `🤖 Бот запущен. Выполняю начальную синхронизацию задач...`, а после её окончания — `✅ Бот успешно синхронизировал все задачи! Приятного использования.`.

## 🔒 Безопасность и Конфиденциальность

В соответствии с правилами безопасности `.gitignore` исключает следующие файлы из индексации Git. **Никогда не публикуйте их в публичных репозиториях:**
*   `.env` — конфиденциальные токены Telegram-бота и ID чатов.
*   `credentials.json` — ключи доступа к вашему Google Cloud приложению.
*   `*token.json` (`Person1_token.json`, `Person2_token.json`, `token.json`) — персональные токены доступа к задачам пользователей.
*   `tasks_state.db` — локальная база данных задач.
